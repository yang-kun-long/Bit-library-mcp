# 技术实现日志 (Tech Journal)

## 2026-04-01: 核心通信协议修复与增强提取功能

### 1. WebSocket 通信协议对齐 (Critical)
**问题描述**: 
AI 客户端通过 MCP 服务器发送任务时，消息被封装在 `{type: 'TASK', payload: { type: '...', ... }}` 结构中。插件背景脚本之前无法识别这种双层结构，导致任务无法派发并触发 "Task execution timeout"。

**修复方案**:
在 `background.js` 的 `handleMessage` 中增加了自动解包逻辑：
```javascript
if (msg.type === 'TASK') {
  type = payload.type;
  // 保持 taskId，payload 指向内部数据对象
}
```

### 2. 论文元数据提取能力增强
**目标**: 抓取超星发现/智真系统详情页的深度学术信息。

**新增字段**:
- **作者单位 (Affiliation)**: 支持多机构提取。
- **基金项目 (Funding)**: 提取具体的科研基金信息。
- **出版信息 (Pub Info)**: 自动合并卷号、期号、页码。
- **核心收录 (Indexing)**: 识别核心期刊状态（核/源等）。
- **评价指标 (Impact Factor)**: 提取期刊影响因子。
- **标准引文 (Citation)**: 通过 `dxid` 异步获取官方引文格式。

**技术实现点**:
- **模糊匹配**: 不再依赖固定 HTML 路径，通过关键词（如 "单位"、"基金"）定位数据元素。
- **Iframe 穿透**: 递归搜索页面内所有子框架，支持嵌套在 Iframe 中的内容提取。
- **DOM 容错**: 同时支持 `dl`, `dt`, `dd` 和 `div`, `span` 等多种布局变体。

### 3. 调试与可视化优化
- **活动窗口**: 详情页抓取时将标签页设为 `active: true`，方便用户观察进度或手动登录。
- **状态保留**: 抓取失败时**不再自动关闭标签页**，并输出详细错误日志到控制台，极大降低了排错成本。
- **控制台前缀**: 统一使用 `[MCP]` 和 `[Injected]` 前缀标记日志来源。

### 4. 环境连接检查
- 修复了 `popup.js` 检查连接状态时触发的 `Receiving end does not exist` 错误，通过在 `background.js` 增加消息监听器实现稳定性。
